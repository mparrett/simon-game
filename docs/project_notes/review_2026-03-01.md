# Code Review - 2026-03-01

## Issues Identified

### Stale closure in handleColorClick
Plain function (not useCallback) captures `playerSequence` from closure rather than setter callback. Rapid clicks could theoretically race against state updates. Low risk for Simon game pacing.

### Fragile playSequence useEffect (line 296-300)
Effect depends on `[sequence, isPlaying, playSequence]`, and `playSequence` itself depends on `sequence` via useCallback. Tight coupling — changes to playSequence dependencies could trigger unintended replays.

### Repetitive sound functions (~100 lines)
`playSound`, `playStartButtonSound`, `playCountdownSound`, `playTimeoutSound` (lines 53-148) all follow identical create-oscillator/connect/ramp/stop pattern with different parameters. Should be a single parameterized helper.

### Timer clears on any click, not just correct (line 210)
`handleColorClick` clears timeLeft on any click including wrong ones. Not a bug (endGame also clears timer) but confusing flow.

### Unused dependencies in package.json
`@hookform/resolvers`, `recharts`, `lucide-react`, `@radix-ui/react-alert-dialog`, `@radix-ui/react-toast` — none used in the component.

## Actions Taken
- Committed UI polish changes (current score column, button glow, always-visible history)
- Consolidated sound functions into single parameterized helper
